# Embedded Firmware Interview Coding Topics

The questions are often more practical than those asked by companies
focused purely on competitive programming.

## 1. Arrays (Very Common)

**Examples:** - Find the first non-repeating element. - Move all zeros
to the end. - Remove duplicates from a sorted array. - Rotate an array
by K positions. - Find the second largest element. - Find missing
number. - Maximum subarray sum (Kadane's Algorithm). - Merge two sorted
arrays. - Find majority element. - Product of array except self.

**Difficulty:** Easy to Medium

## 2. Strings (Very Common)

**Examples:** - Reverse words in a string. - Check if two strings are
anagrams. - Compress a string (Run-Length Encoding). - Longest substring
without repeating characters. - Reverse only alphabets. - Check
palindrome ignoring spaces. - Count occurrences of a substring.

**Difficulty:** Medium

## 3. Hash Map / Hash Set

**Examples:** - Two Sum - First duplicate element - Frequency counting -
Intersection of two arrays - Find unique sensor IDs

**Difficulty:** Medium

## 4. Sliding Window

**Examples:** - Maximum sum of K consecutive elements - Moving average -
Longest substring with K distinct characters - Minimum window substring

These are very relevant for firmware because they resemble processing
continuous streams of sensor or communication data.

## 5. Linked List

**Examples:** - Reverse a linked list - Detect a loop (Floyd's Cycle
Detection) - Find the middle node - Merge two sorted linked lists -
Remove the Nth node from the end

## 6. Stack and Queue

**Examples:** - Balanced parentheses - Implement a queue using stacks -
Implement a stack using queues - Next greater element - Min stack

## 7. Binary Tree

**Examples:** - Tree traversal (Preorder, Inorder, Postorder) -
Level-order traversal - Height of a tree - Lowest Common Ancestor
(LCA) - Check if a tree is balanced

## 8. Binary Search

**Examples:** - Search in a rotated sorted array - Find first and last
occurrence - Find square root - Peak element - Lower bound / Upper bound

## 9. Bit Manipulation (Very Common for Embedded)

**Examples:** - Count set bits - Check power of two - Reverse bits -
Find the only non-repeating number - Swap two numbers without a
temporary variable - Set, clear, toggle, and check a bit - Count
leading/trailing zeros (conceptually)

## 10. C/C++ Questions

These are especially important for embedded roles.

**Examples:** - Pointer arithmetic - Function pointers - `const`
correctness - `volatile` - `static` - Memory layout - Structures
vs. unions - Structure padding - Dynamic memory allocation - Virtual
functions (C++) - References vs. pointers - Copy constructor - Move
constructor (for experienced roles)

## 11. Embedded-Oriented Coding

These questions are very common because they simulate real firmware
tasks.

**Examples:** - Implement a circular buffer (ring buffer) - Implement a
fixed-size queue - Design a software timer - Write a debounce algorithm
for a button - Parse UART packets - Implement a state machine - CRC
calculation - Checksum validation - FIFO implementation - Event
scheduler

## 12. RTOS-Based Coding

**Examples:** - Producer-consumer problem - Thread-safe queue -
Synchronization using mutexes or semaphores - Deadlock scenarios -
Priority inversion discussion - Task scheduling concepts
